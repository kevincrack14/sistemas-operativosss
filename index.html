Sistemas Operativos

BREVE HISTORIA DE LOS SISTEMAS OPERATIVOS: DESDE UNIX HASTA WINDOWS 11

Los sistemas operativos han evolucionado junto con el desarrollo de la informática moderna, facilitando la interacción entre usuario, hardware y software. Uno de los pilares fundamentales fue UNIX, creado en 1969 por Ken Thompson y Dennis Ritchie, cuyo diseño modular, multitarea y multiusuario influyó en numerosos sistemas posteriores como Linux y macOS.

En 1991 surgió Linux, desarrollado por Linus Torvalds, impulsando el software libre y convirtiéndose en un sistema clave para servidores, la nube y dispositivos móviles mediante Android.

Apple adoptó tecnologías basadas en UNIX para crear Mac OS X (hoy macOS), combinando estabilidad, seguridad y una experiencia de usuario optimizada. Por su parte, Microsoft Windows evolucionó desde MS-DOS hasta Windows 11, pasando por versiones clave como Windows 95, NT y XP. Windows se consolidó como el sistema dominante en computación personal, integrando mejoras en seguridad, interfaz y compatibilidad con tecnologías modernas como la nube y Linux (WSL). En conclusión, la evolución de los sistemas operativos refleja el avance tecnológico hacia sistemas más portables, seguros, modulares y orientados a la conectividad global.

CONCEPTO DE HILOS

Un hilo (thread) es la unidad más pequeña de ejecución dentro de un proceso, un proceso es un programa en ejecución (por ejemplo, un navegador). Un hilo es una tarea específica que se ejecuta dentro de ese proceso.

🧠 Ejemplos sencillos:

Un hilo carga la página web.

Otro hilo reproduce un video.

Otro hilo descarga un archivo.

Otro hilo mantiene activa la interfaz.

Todo ocurre dentro del mismo proceso, pero gracias a los hilos puede hacer varias cosas simultáneamente.

🤔¿Por qué existen los hilos?🤔

Mejor uso del procesador

Mayor rapidez en tareas concurrentes

Mejor experiencia de usuario

Paralelismo en sistemas con varios núcleos

🧐Tipos de hilos🧐

👤Hilos a nivel de usuario👤

Gestionados por una biblioteca del programa.

El sistema operativo no los maneja directamente.

🖥️Hilos a nivel de núcleo🖥️ (kernel)

Administrados por el sistema operativo.

Más seguros y estables.

Permiten verdadero paralelismo en CPU multinúcleo.

IMPORTANCIA DE LA MEMORIA

La memoria es un componente fundamental en el funcionamiento de cualquier sistema, especialmente en los sistemas informáticos, ya que permite almacenar, acceder y procesar la información necesaria para que el sistema pueda operar correctamente. En términos generales, la memoria actúa como el espacio donde se guardan temporal o permanentemente los datos y las instrucciones que el procesador necesita para realizar distintas tareas. Sin este elemento, el funcionamiento de una computadora o de cualquier sistema digital sería prácticamente imposible, debido a que el procesador no tendría acceso a la información requerida para ejecutar operaciones. Por esta razón, la memoria se considera una parte esencial de la arquitectura de los sistemas informáticos modernos.

Uno de los aspectos más importantes de la memoria es su función de almacenamiento de información. La memoria permite guardar tanto datos como instrucciones que los programas necesitan para ejecutarse. Cada vez que se abre una aplicación, se escribe un documento o se ejecuta cualquier programa, la información necesaria se almacena temporalmente en la memoria para que el procesador pueda utilizarla. Este almacenamiento facilita que los datos estén disponibles de manera inmediata, evitando que el sistema tenga que buscarlos constantemente en dispositivos de almacenamiento más lentos, como los discos duros o las unidades de estado sólido. De esta manera, la memoria contribuye a que los procesos se realicen de forma más eficiente y rápida, ya que el acceso a la información es mucho más veloz cuando se encuentra en la memoria principal.

Otro aspecto clave de la memoria es su papel en la ejecución de programas. Cuando un usuario inicia un programa en su computadora, el sistema operativo carga las instrucciones y los datos del programa en la memoria. A partir de ese momento, el procesador puede acceder rápidamente a esa información para ejecutar cada una de las instrucciones que componen el programa. Este proceso ocurre continuamente mientras el programa está en funcionamiento, permitiendo que el sistema responda a las acciones del usuario de manera ágil. Si no existiera la memoria, el procesador tendría que acceder directamente a dispositivos de almacenamiento más lentos para cada instrucción, lo que haría que el sistema funcionara de manera extremadamente lenta o incluso imposible de utilizar en la práctica.

La memoria también tiene una influencia directa en la velocidad y el rendimiento general del sistema. Cuanto mayor sea la capacidad de memoria y más rápida sea su velocidad de acceso, mejor será el desempeño del equipo. Una memoria eficiente permite que el sistema acceda a los datos con mayor rapidez, reduciendo los tiempos de espera y permitiendo que las tareas se realicen de forma más fluida. Esto es especialmente importante en entornos donde se ejecutan aplicaciones exigentes, como programas de diseño, edición de video, simulaciones o videojuegos. En estos casos, contar con suficiente memoria RAM puede marcar una gran diferencia en la experiencia del usuario, ya que permite que el sistema procese grandes volúmenes de información sin presentar ralentizaciones significativas.

Además, la memoria juega un papel fundamental en la capacidad del sistema para manejar múltiples tareas simultáneamente. En los sistemas modernos es común que los usuarios tengan varias aplicaciones abiertas al mismo tiempo, como navegadores web, procesadores de texto, reproductores de música o programas de diseño. La memoria permite que cada uno de estos programas tenga un espacio donde almacenar sus datos y procesos en ejecución. Gracias a esto, el sistema puede alternar entre diferentes tareas de manera eficiente sin que el usuario tenga que cerrar un programa para abrir otro. Esta capacidad de multitarea es una de las características más importantes de los sistemas operativos actuales, ya que permite mejorar la productividad y facilitar el trabajo en diferentes actividades al mismo tiempo. Finalmente, la memoria es esencial para el funcionamiento del sistema operativo.

El sistema operativo utiliza la memoria para gestionar los recursos del sistema, administrar los procesos que se están ejecutando y coordinar la comunicación entre el hardware y el software. También se encarga de asignar espacio de memoria a los diferentes programas y de controlar cómo se utilizan esos recursos para evitar conflictos o errores. Además, el sistema operativo implementa diversos mecanismos para optimizar el uso de la memoria, como la administración de procesos y la gestión eficiente de los recursos disponibles. De esta manera, la memoria no solo sirve para almacenar datos, sino que también permite que el sistema opere de forma organizada, eficiente y estable, garantizando el correcto funcionamiento de todos los componentes del sistema informático.

SEGMENTACIÓN

La segmentación es una técnica de administración de memoria que divide los programas en diferentes partes llamadas segmentos. Cada segmento corresponde a una unidad lógica del programa, como el código, las variables, las funciones o la pila de ejecución. Estos segmentos pueden tener diferentes tamaños y se almacenan en distintas posiciones de la memoria. La segmentación permite organizar mejor la memoria y facilita la protección y el acceso a los datos, ya que cada segmento puede tener permisos específicos. Por ejemplo, algunos segmentos pueden ser solo de lectura mientras que otros pueden ser modificables. Este método ayuda a mejorar la seguridad, la organización de los programas y la eficiencia en el uso de la memoria, ya que permite que el sistema operativo gestione cada parte del programa de manera independiente.

MEMORIA VIRTUAL

La memoria virtual es una técnica utilizada por los sistemas operativos que permite que los programas utilicen más memoria de la que físicamente tiene el computador. Esto se logra mediante el uso del almacenamiento secundario, como el disco duro o las unidades de estado sólido, para simular una extensión de la memoria principal (RAM). Cuando la memoria RAM se llena, el sistema operativo mueve temporalmente algunos datos que no se están utilizando al disco, liberando espacio para los procesos activos. Gracias a este mecanismo, los programas pueden ejecutarse aunque requieran más memoria de la disponible físicamente. La memoria virtual mejora la estabilidad del sistema, permite ejecutar múltiples aplicaciones al mismo tiempo y evita que los programas se detengan por falta de memoria, contribuyendo así al rendimiento y eficiencia general del sistema operativo.

PAGINACIÓN

La paginación es otra técnica utilizada por los sistemas operativos para administrar la memoria de manera más eficiente. En este método, la memoria se divide en bloques de tamaño fijo llamados páginas, mientras que la memoria física se divide en bloques del mismo tamaño llamados marcos de página. Cuando un programa se ejecuta, sus páginas se cargan en los marcos disponibles de la memoria física. El sistema operativo utiliza una tabla de páginas para llevar un registro de dónde se encuentra cada página del programa en la memoria. Este sistema permite que las páginas de un programa no tengan que estar necesariamente juntas en la memoria física, lo que facilita el uso más eficiente del espacio disponible. En términos de rendimiento y eficiencia del sistema operativo, la paginación tiene una ventaja importante porque evita el problema de la fragmentación externa, que ocurre cuando la memoria se divide en muchos espacios pequeños inutilizables. Al trabajar con bloques de tamaño fijo, el sistema puede aprovechar mejor la memoria disponible y administrar los procesos de manera más ordenada. Sin embargo, puede existir fragmentación interna cuando una página no utiliza completamente el espacio asignado. Aun así, la paginación es ampliamente utilizada porque simplifica la gestión de la memoria y permite que los sistemas operativos ejecuten múltiples programas de forma eficiente.
